MOTIVATION
WHY QUANTUM-SAFE CHANNELS MATTER FOR AI AGENTS
Today's AI agents communicate over TLS channels secured by RSA and ECDH — algorithms that a fault-tolerant quantum computer will break using Shor's algorithm, potentially within 5–15 years.

Adversaries are already collecting encrypted traffic today to decrypt later — known as the harvest-now, decrypt-later attack. Agent networks handling sensitive data need quantum-safe channels now, not at Q-Day.

This POC demonstrates what that looks like in practice — two live AI agents running a full post-quantum handshake, exchanging encrypted and signed messages, and surviving a simulated quantum attack.
TECHNOLOGY
THREE ALGORITHMS, ONE SECURE CHANNEL
🔑
NIST FIPS 203
ML-KEM-768
Key encapsulation mechanism based on Module-LWE lattice problem. Establishes a shared secret between agents. Resistant to Shor's algorithm. Security Level 3.
🪪
NIST FIPS 204
ML-DSA-65
Digital signature algorithm for agent identity and message signing. Every message is signed before encryption — guaranteeing authenticity and non-repudiation.
🔐
SYMMETRIC ENCRYPTION
AES-256-GCM
Authenticated encryption for message payload. Per-message IV derived from session. Key derived from ML-KEM-768 shared secret. Zero IV reuse guaranteed.
HOW IT WORKS
FROM HANDSHAKE TO SECURE MESSAGE
01
KEY GENERATION
Each agent generates an ML-KEM-768 keypair for key establishment and an ML-DSA-65 keypair for identity signing.
02
SHARED SECRET
Agents run ML-KEM-768 encapsulation. A shared secret is derived. An AES-256-GCM session key is derived from that secret.
03
SIGN THEN ENCRYPT
Each message is ML-DSA-65 signed first, then AES-256-GCM encrypted. A message counter prevents replay attacks.
04
AUDIT LOGGED
Every handshake and message event is written to a tamper-evident audit trail with timestamps and signature references.
05
ATTACK SIMULATED
Shor's algorithm is simulated against the session. Classical RSA breaks instantly. ML-KEM-768 lattice keys hold.
